class agents.config.LLMConfig#

Bases: agents.config.ModelComponentConfig

Configuration for the Large Language Model (LLM) component.

It defines various settings that control how the LLM component operates, including whether to enable chat history, retreival augmented generation (RAG) and more.

Parameters:

  • enable_rag (bool) ‚Äì Enables or disables Retreival Augmented Generation.

  • collection_name (Optional[str]) ‚Äì The name of the vectordb collection to use for RAG.

  • distance_func (str) ‚Äì The distance metric used for nearest neighbor search for RAG. Supported values are ‚Äúl2‚Äù, ‚Äúip‚Äù, and ‚Äúcosine‚Äù.

  • n_results (int) ‚Äì The maximum number of results to return for RAG. Defaults to 1. For numbers greater than 1, results will be concatenated together in a single string.

  • chat_history (bool) ‚Äì Whether to include chat history in the LLM‚Äôs prompt.

  • history_reset_phrase (str) ‚Äì Phrase to reset chat history. Defaults to ‚Äòchat reset‚Äô

  • history_size (int) ‚Äì Number of user messages to keep in chat history. Defaults to 10

  • temperature (float) ‚Äì Temperature used for sampling tokens during generation. Default is 0.8 and must be greater than 0.0.

  • max_new_tokens (int) ‚Äì The maximum number of new tokens to generate. Default is 100 and must be greater than 0.

  • stream (bool) ‚Äì Publish the llm output as a stream of tokens, useful when sending llm output to a user facing client or to a TTS component. Cannot be used in conjunction with tool calling. Default is false

  • break_character (str) ‚Äì A string character marking that the output thus far received in a stream should be published. This parameter only takes effect when stream is set to True. As stream output is received token by token, it is useful to publish full sentences instead of individual tokens as the components output (for example, for downstream text to speech conversion). This value can be set to an empty string to publish output token by token. Default is ‚Äò.‚Äô (period)

  • response_terminator (str) ‚Äì A string token marking that the end of a single response from the model. This token is only used in case of a persistent clients, such as a websocket client and when stream is set to True. It is not published. This value cannot be an empty string. Default is ‚Äò<>‚Äô

Example of usage:

config = LLMConfig(enable_rag=True, collection_name="my_collection", distance_func="l2")

class agents.config.SpeechToTextConfig#

Bases: agents.config.ModelComponentConfig

Configuration for a Speech-To-Text component.

This class defines the configuration options for speech transcription, voice activity detection, wakeword detection, and audio streaming.

Transcription

Parameters:

  • initial_prompt (str or None) ‚Äì Optional initial prompt to guide transcription (e.g. speaker name or topic). Defaults to None.

  • language (str) ‚Äì Language code for transcription (e.g. ‚Äúen‚Äù, ‚Äúzh‚Äù). Must be one of the supported language codes. Defaults to ‚Äúen‚Äù.

  • max_new_tokens (int or None) ‚Äì Maximum number of tokens to generate. If None, no limit is applied. Defaults to None.

Voice Activity Detection (VAD)

Parameters:

  • enable_vad (bool) ‚Äì Enable VAD to detect when speech is present in audio input. Requires onnxruntime and silero-vad model. Defaults to False.

  • device_audio (Optional[int]) ‚Äì Audio input device ID. Only used if enable_vad is True. Defaults to None.

  • vad_threshold (float) ‚Äì Threshold above which speech is considered present. Only used if enable_vad is True. Range: 0.0‚Äì1.0. Defaults to 0.5.

  • min_silence_duration_ms (int) ‚Äì Minimum silence duration (ms) before it‚Äôs treated as a pause. Only used if enable_vad is True. Defaults to 300.

  • speech_pad_ms (int) ‚Äì Silence padding (ms) added to start and end of detected speech regions. Only used if enable_vad is True. Defaults to 30.

  • speech_buffer_max_len (int) ‚Äì Max length of speech buffer in ms. Only used if enable_vad is True. Defaults to 30000.

  • device_vad (str) ‚Äì Device for VAD (‚Äòcpu‚Äô or ‚Äògpu‚Äô). Only used if enable_vad is True. Defaults to ‚Äòcpu‚Äô.

  • ncpu_vad (int) ‚Äì Number of CPU cores to use for VAD (if device_vad is ‚Äòcpu‚Äô). Defaults to 1.

Wakeword Detection

Parameters:

  • enable_wakeword (bool) ‚Äì Enable detection of a wakeword phrase (e.g. ‚ÄòHey Jarvis‚Äô). Requires enable_vad to be True. Defaults to False.

  • wakeword_threshold (float) ‚Äì Minimum confidence score to trigger wakeword detection. Only used if enable_wakeword is True. Defaults to 0.6.

  • device_wakeword (str) ‚Äì Device for Wakeword Detection (‚Äòcpu‚Äô or ‚Äògpu‚Äô). Only used if enable_wakeword is True. Defaults to ‚Äòcpu‚Äô.

  • ncpu_wakeword (int) ‚Äì Number of CPU cores for Wakeword Detection (if device_wakeword is ‚Äòcpu‚Äô). Defaults to 1.

Streaming

Parameters:

  • stream (bool) ‚Äì Send audio as a stream to a persistent client (e.g., websockets). Requires enable_vad to be True. Useful for real-time transcription. Defaults to False.

  • min_chunk_size (int) ‚Äì Audio chunk size in ms to send when streaming. Requires stream to be True. Must be > 100 ms. Defaults to 2000.

Model Paths

Parameters:

  • vad_model_path (str) ‚Äì Path or URL to VAD ONNX model. Defaults to the Silero VAD model URL.

  • melspectrogram_model_path (str) ‚Äì Path or URL to melspectrogram model used in wakeword detection. Defaults to openWakeWord model URL.

  • embedding_model_path (str) ‚Äì Path or URL to audio embedding model for wakeword detection. Defaults to openWakeWord model URL.

  • wakeword_model_path (str) ‚Äì Path or URL to wakeword ONNX model (e.g. ‚ÄòHey Jarvis‚Äô). Defaults to a pretrained openWakeWord model. For custom models, see: dscripka/openWakeWord

Example

Example usage:

config = SpeechToTextConfig(
    enable_vad=True,
    enable_wakeword=True,
    vad_threshold=0.5,
    wakeword_threshold=0.6,
    min_silence_duration_ms=1000,
    speech_pad_ms=30,
    speech_buffer_max_len=8000,
)

class agents.config.TextToSpeechConfig#

Bases: agents.config.ModelComponentConfig

Configuration for a Text-To-Speech component.

This class defines the configuration options for a Text-To-Speech component.

Parameters:

  • play_on_device (bool) ‚Äì Whether to play the audio on available audio device (default: False).

  • device (int) ‚Äì Optional device id (int) for playing the audio. Only effective if play_on_device is True (default: None).

  • stream_to_ip (Optional[str]) ‚Äì If set, streams the audio to this IP address via UDP instead of playing locally. Requires play_on_device to be True.

  • stream_to_port (Optional[int]) ‚Äì The target port for UDP streaming. Must be set if stream_to_ip is set.

  • buffer_size (int) ‚Äì Size of the buffer for playing audio on device. Only effective if play_on_device is True (default: 20).

  • block_size (int) ‚Äì Size of the audio block to be read for playing audio on device. Only effective if play_on_device is True (default: 1024).

  • thread_shutdown_timeout (int) ‚Äì Timeout to shutdown a playback thread, if data is not received for more than a certain number of seconds. Only effective if play_on_device is True (default: 5 seconds).

  • stream ‚Äì Stram output when used with WebSocketClient. Useful when model output is large and broken into chunks by the server. (default: True).

Example of usage for local playback:

config = TextToSpeechConfig(play_on_device=True)

Example of usage for UDP streaming:

config = TextToSpeechConfig(play_on_device=True, stream_to_ip="192.168.1.100", stream_to_port=12345)

Parameters:

  • distance_func (str) ‚Äì The function used to calculate distance from route samples in vectordb. Can be one of ‚Äúl2‚Äù (L2 distance), ‚Äúip‚Äù (Inner Product), or ‚Äúcosine‚Äù (Cosine similarity). Default is ‚Äúl2‚Äù.

class agents.config.VideoMessageMakerConfig#

Bases: agents.ros.BaseComponentConfig

Configuration parameters for a video message maker component.

Parameters:

  • min_video_frames (int) ‚Äì The minimum number of frames in a video segment. Default is 15, assuming a 0.5 second video at 30 fps.

  • max_video_frames (int) ‚Äì The maximum number of frames in a video segment. Default is 600, assuming a 20 second video at 30 fps.

  • motion_estimation_func (Optional[str]) ‚Äì The function used for motion estimation. Can be one of ‚Äúframe_difference‚Äù or ‚Äúoptical_flow‚Äù. Default is None.

  • threshold (float) ‚Äì The threshold value for motion detection. A float between 0.1 and 5.0. Default is 0.3.

  • flow_kwargs ‚Äì Additional keyword arguments for the optical flow algorithm. Default is a dictionary with reasonable values.

Example of usage:

# or
config = VideoMessageMakerConfig(min_video_frames=30, motion_estimation_func="optical_flow", threshold=0.5)
